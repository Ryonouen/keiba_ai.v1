# ダッシュボード タブ再構成 設計書

**日付:** 2026-04-08  
**ゴール:** 「レース分析」と「集計レポート」を明確に分離し、競馬場・日付ナビゲーションを整理する

---

## 概要

現状の2タブ（当日 / 履歴）を4タブに刷新する。

```
[🏇 レース分析] [📊 日次レポート] [📅 月次レポート] [📆 年次レポート]
```

---

## Tab 1: レース分析

### ナビゲーション構造

```
日付サブタブ: [📅 今日(MM/DD)] [土(MM/DD)] [日(MM/DD)]
  └ 競馬場サブタブ: [🏟 中山] [🏟 阪神] ...（開催場のみ表示）
       └ レースカード一覧（既存の _render_race_cards() をそのまま使用）
```

### 日付の決定ロジック

| 今日の曜日 | 表示する土 | 表示する日 |
|-----------|-----------|-----------|
| 月〜金 | 直近の次の土曜 | 直近の次の日曜 |
| 土 | 今日（土） | 明日（日） |
| 日 | 昨日（土） | 今日（日） |

- 60秒自動更新は「今日」タブのみに適用（`@st.fragment(run_every=60)`）
- 競馬場タブ: 選択日の `load_races_for_date()` 結果を競馬場ごとにグループ化し、データがある場のみタブを生成
- データなし（土日でパイプライン未実行など）の場合は「データがありません」メッセージを表示

---

## Tab 2: 日次レポート

- 日付セレクタ（`get_available_dates()` で取得できる日付のみ）
- 選択日の KPI（投資額・回収額・ROI・的中数/総買い目数）
- 券種別集計テーブル
- その日のレースカード一覧（参照用、折りたたみ可）

---

## Tab 3: 月次レポート

- 年月セレクタ（利用可能な日付から年月リストを生成）
- 月合計 KPI
- 日別 ROI 推移グラフ（折れ線）
- 日別明細テーブル（日付・投資・回収・ROI・的中/総買い目）

---

## Tab 4: 年次レポート

- 年セレクタ（利用可能な日付から年リストを生成）
- 年合計 KPI
- 月別 ROI 推移グラフ（折れ線）
- 月別明細テーブル（年月・投資・回収・ROI・的中/総買い目）

### keiba_app.py の変更

- `_tab_today()` → `_tab_race_analysis()` に改名・拡張
  - 日付サブタブ（今日/土/日）を追加
  - 60秒自動更新は今日タブのみ `@st.fragment` で適用
  - 各日付タブ内で競馬場サブタブを生成
- `_tab_history()` → 削除し、以下3関数に分割
  - `_tab_daily()`: 日次レポート
  - `_tab_monthly()`: 月次レポート
  - `_tab_yearly()`: 年次レポート
- メイン: `st.tabs(["🏇 レース分析", "📊 日次レポート", "📅 月次レポート", "📆 年次レポート"])`
